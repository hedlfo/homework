OSI 7계층

7계층 -애플리케이션 계층(Application Layer)
6계층 -프레젠테이션 계층(Presentation Layer)
5계층 -세션 계층(Session Layer)
4계층 -트랜스포트 계층(Transport Layer)
3계층 -네트워크 계층(Network Layer)
2계층 -데이터 링크 계층(Data Link Layer)
1계층 -피지컬 계층(Physical Layer)


1계층 (피지컬 계층)
: 물리적 연결과 관련된 정보 정의
주로 전기 신호 전달.
주소 개념 x

2계층(데이터 링크 계층)
주소 개념o
에러를 탐지하는 역할만 수행.
2계층에서 동작하는 네트워크 구성 요소는
1-네트워크 인터페이스 카드
2-스위치
, 중요 특징: MAC주소 라는 주소 체계가 있음.

3계층(네트워크 계층)
IP와 같은 논리적인 주소가 정의됨.

(2계층-MAC주소(물리적)/3계층-IP주소(논리적))
3계층에서 동작하는 네트워크 구성 요소는
라우터, 
라우터는 IP주소를 사용해 최적의 경로를 찾아주고 해당 경로로 패킷을 전송하는 역할을 함.

4계층(트랜스포트 계층)
4계층은 실제로 해당 데이터들이 정상적으로 잘 보내지는지 확인하는 역할을 함.
*패킷 네트워크-중간에 패킷이 유실되거나 순서가 바뀌는 경우가 생길 수 있음.
이를 바로잡아주는 역할을 4계층에서 담당.
패킷에 보내는 순서와 받는 순서를 적어 유실을 방지한다.
패킷에 보내는 순서를 명시한 것-시퀀스 번호
받는 순서를 명시한 것-ACK번호
그리고 장치내 앱을 구분하기 위해 -포트 번호를 사용
4계층에서 동작하는 장비로는
로드 밸런서, 방화벽이 있음.

5계층(세션 계층)
양 끝단의 응용프로세스가 연결을 성립하도록 도와주고 연결이 안정적으로 유지되도록
관리하고 작업 완료 후에는 연결을 끊음.
TCP/IP세션을 만들고 없애는 책임을 짐.
에러 복구와 재전송도 수행.

6계층(프레젠테이션 계층)
표현 방식이 다른 앱이나 시스템간통신을 돕기 위해 통일된 구문 형식으로 변환하는 기능 수행

7계층(애플리케이션 계층)
네트워크 소프트웨어의 UI부분이나 사용자 입출력부분을 정의하는 역할.
애플리케이션 계층의 프로토콜은 많은 종류가 있으나
대표적 프로토콜로 FTP, SMTP, HTTP, TELNET 등이 있음.
